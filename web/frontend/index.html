About This Project


Project Developed by OM SETH | View Project on GitHub


This is a wireless sensor network simulator built to show how Voronoi-based scheduling can save energy by turning redundant nodes into standby backup nodes while keeping sensing coverage high.

Main features

  • Voronoi-based node scheduling
  • Backup node concept for energy saving
  • Round-wise fault and recovery simulation
  • Algorithm comparison on the same field
  • AI-assisted scheduling using a local ML model
  • Experiment and CSV export support

Current working

  • Generate a node field with a fixed seed
  • Schedule nodes using Voronoi or comparison algorithms
  • Measure coverage, energy saved, and lifetime gain
  • Inject failures round by round
  • Recover coverage using backup nodes
  • Compare both initial and final results clearly

Technology stack

  • FastAPI backend
  • HTML, CSS, JavaScript frontend
  • Plotly for charts
  • NumPy, SciPy, and Shapely for simulation geometry
  • scikit-learn for ML
  • Local file-based storage for ML datasets and models

AI / ML part

  • Local RandomForest classifier predicts node ON/OFF states
  • Local regressor estimates run-level metrics
  • Retraining appends new synthetic WSN data to stored datasets
  • Main page only gives quick access; full ML details stay in ML Lab

Voronoi-Threshold

Uses the bounded Voronoi responsibility area of each node. Nodes with very small unique area are turned into backup nodes.

Random-Same-OFF

Turns off the same number of nodes as Voronoi, but randomly. It is used as a weak baseline.

Random-Greedy

Tries random removal while checking coverage so the result stays closer to the target coverage.

Greedy-Coverage

Keeps only nodes that add useful coverage, so it is a stronger coverage-first heuristic.

AI-Based

Uses the trained local ML model to assist scheduling. If no model is present, the system safely falls back to Voronoi.

Sources

  • Scheduling Nodes in Wireless Sensor Networks: A Voronoi Approach — core research basis for scheduling logic
  • FastAPI documentation — backend API structure
  • Plotly documentation — charts and graph rendering
  • NumPy, SciPy, and Shapely documentation — geometry and numerical processing
  • scikit-learn documentation — RandomForest classifier and regressor
  • All simulation metrics, comparison outputs, and ML history shown in the dashboard come from this project’s own implemented logic and stored local datasets.
